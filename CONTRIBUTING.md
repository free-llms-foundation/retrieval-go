# Contributing

Thanks for your interest in contributing to `retrieval-go`.

## Getting started

- Fork the repo
- Create a feature branch
- Make your changes
- Open a pull request

## Development

- Use a recent Go toolchain
- Run:

```bash
go test ./...
```

## Pull requests

- Keep PRs focused and small when possible
- Prefer backward-compatible API changes
- Add tests or fixtures when changing parsing behavior
- Clearly describe:
  - what was changed
  - why it was changed
  - how to verify the change

## Code of Conduct

By participating, you are expected to uphold the [Code of Conduct](./CODE_OF_CONDUCT.md).
